# McKinsey 箭头流程图 (Chevron Process Flow)

**适用场景：**
展示线性过程、阶段性计划、价值链或时间轴。这是咨询报告中最基础也最常用的图表之一。

**视觉风格参考：**
McKinsey3.pdf, Slide 42 (Customer decision journey)

**Prompt:**
请生成一个基于箭头（Chevron）的水平流程图。

1.  **主要形状：**
    *   使用一系列向右指的宽箭头（Chevron形状）横向排列。
    *   箭头之间保留微小的间隙。
    *   **颜色：** 使用深海军蓝作为主色，或者交替使用深蓝和深灰。如果是强调某个阶段，将该阶段设为亮蓝色。

2.  **内容层级：**
    *   **箭头内：** 放置阶段名称（如 "Exploratory", "Development"），字体白色加粗。
    *   **箭头下方：** 对应每个箭头，列出详细的子项，如 "Activities", "Timeline", "Key Outputs"。这些内容应与上方的箭头严格对齐。

3.  **连接线：**
    *   如果在箭头下方有多个维度的信息（如时间、活动），使用水平的细灰线将这些维度分隔开。

# McKinsey 漏斗图 (Funnel Chart - Vertical & Horizontal)

**适用场景：**
**垂直漏斗：** 市场规模估算（TAM/SAM/SOM）、销售漏斗转化。
**水平漏斗：** 战略筛选过程（从长名单到短名单）。

**视觉风格参考：**
McKinsey2.pdf, Slide 26 (Vertical), Slide 27 (Horizontal)

**Prompt 1: 垂直漏斗 (Market Sizing Funnel)**
请生成一个垂直漏斗图，用于展示层级递减的数据。
1.  **形状：** 一个倒梯形（漏斗），被水平切割成3-4层。
2.  **颜色：** 整个漏斗使用深蓝色。层与层之间用细白线隔开。
3.  **标签：**
    *   **漏斗内：** 显示数值（如 "$XX M"）或百分比。
    *   **漏斗右侧：** 使用引线（Leader lines）指向每一层，并在右侧写上详细的定义文本（如 "Total Spend", "Addressable Market"）。

**Prompt 2: 水平筛选漏斗 (Screening Process)**
请生成一个横向的筛选流程图，形状类似一个巨大的向右箭头。
1.  **形状：** 一个巨大的长箭头，被垂直线分割成3个阶段。
2.  **概念：** 表现从左侧的"大量选项"（All value domains）经过筛选，变成右侧的"最终优先级"（Priorities）。
3.  **内部内容：**
    *   阶段1：宽头，"High value"。
    *   阶段2：中间，"Feasible"。
    *   阶段3：尖头，"Win"。
4.  **底部支撑：** 在箭头下方，对应每个阶段列出具体的筛选标准（Criteria）或考虑因素列表。

# McKinsey 甘特图 (Gantt Chart / Timeline)

**适用场景：**
项目计划、实施路线图、周计划。

**视觉风格参考：**
McKinsey3.pdf, Slide 41

**Prompt:**
请生成一个专业的甘特图（Gantt Chart）。

1.  **时间轴：**
    *   顶部横向排列时间单位（如 Week 1, Week 2... 或 Jan, Feb...）。
    *   使用垂直的浅灰网格线贯穿整个图表，划分时间区间。

2.  **任务行：**
    *   左侧列出任务名称（Key activities）。
    *   右侧对应时间条（Bars）。

3.  **条形样式（关键）：**
    *   **箭头条：** 时间条的末端设计为箭头状，表示持续性。
    *   **颜色编码：** 使用不同颜色的条形代表不同的责任方（例如：黑色=主要团队，亮蓝=支持团队）。
    *   **标注：** 在条形内部或旁边标注关键里程碑（如 "XX" 或 "Workshop"）。

4.  **阶段划分：**
    *   如果在左侧有大的阶段（如 "Phase 1"），可以用浅蓝色背景块将该阶段的所有相关行包围起来。
