# McKinsey 逻辑树与架构图 (Logic Tree & Structure)

**适用场景：**
问题分解、利润/成本驱动因素分析、组织架构、计算逻辑展示。

**视觉风格参考：**
McKinsey4.pdf, Slide 68 (Calculation Tree), McKinsey4.pdf, Slide 74 (Issue Tree)

**Prompt:**
请生成一个水平方向的逻辑树（Logic Tree）或分解图。

1.  **布局方向：** 从左向右发散。
2.  **节点样式：**
    *   **根节点（左）：** 主要问题或总目标（如 "Profit"）。使用矩形框，深色边框或背景。
    *   **分支（右）：** 使用这种线条连接子节点——线条先水平延伸，再垂直分叉，最后水平连接到子节点（类似电路板走线或组织架构图线条）。不要使用斜线。
3.  **层级：**
    *   至少包含3个层级（总 -> 一级驱动因素 -> 二级细分）。
    *   例如：Profit -> (Revenue - Cost) -> (Price * Vol) - (Fixed + Variable)。
4.  **标注：**
    *   在每个节点旁边可以添加圆圈字母（a, b, c）作为索引，用于对应外部的详细说明。
    *   如果是计算逻辑，可以在节点之间添加小的数学符号圈（+,-, x）。

# McKinsey 维恩图 (Venn Diagram - 3 Circles)

**适用场景：**
展示三个要素的交集，强调核心重叠区域（如"甜蜜点"、"Lighthouse"、"MVP"）。

**视觉风格参考：**
McKinsey2.pdf, Slide 30

**Prompt:**
请生成一个三个圆圈重叠的维恩图（Venn Diagram）。

1.  **圆圈布局：** 三个圆圈呈品字形排列（上一下二，或上二下一）。
2.  **样式：**
    *   **填充：** 使用深色背景（黑色或深蓝）填充圆圈，文字为白色。
    *   **重叠区：** 圆圈重叠的部分不需要变色，而是作为一个整体的视觉中心。
3.  **核心中心：**
    *   在三个圆圈的中心交汇处，放置一个白色的圆形或文本框，这是图表的重点（如 "The Core", "Scale-up Unit"）。
    *   从中心向外引出文字说明。
4.  **外围标注：**
    *   每个大圆圈内部放置图标和标题（如 Business, People, Technology）。
    *   使用大括号或引线在圆圈外部添加更多解释性文本。

# McKinsey 四柱模型 (4-Pillars Layout)

**适用场景：**
展示战略支柱、核心价值观、会议目标或分类建议。

**视觉风格参考：**
McKinsey1.pdf, Slide 10, McKinsey2.pdf, Slide 22

**Prompt:**
请生成一个四列并排的支柱式布局（Pillars Layout）。

1.  **顶部图标：**
    *   每列的顶部放置一个巨大的圆形图标（深色背景，白色线条图标）。图标应代表该列的主题（如地球代表全球，人像代表客户）。
2.  **标题：**
    *   在图标下方使用粗体大号字体书写标题。
3.  **正文内容：**
    *   标题下方对齐排列正文。正文应为左对齐的无衬线字体，行间距适中。
    *   确保四列的高度视觉上保持平衡。
4.  **分隔：**
    *   列与列之间不需要垂直线条，利用大量的留白（Whitespace）进行分隔。
    *   底部可以添加一条横线，并在下方标注统一的注脚或时间分配（如 "15 min"）。
