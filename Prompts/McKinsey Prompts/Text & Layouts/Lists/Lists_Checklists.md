# McKinsey 图标列表 (Icon List / Priorities)

**适用场景：**
列出关键任务、优先事项、建议或步骤。比纯文本列表更具视觉吸引力。

**视觉风格参考：**
McKinsey5.pdf, Slide 94

**Prompt:**
请生成一个带有图标的垂直列表布局。

1.  **列表项结构：**
    *   **图标（左侧）：** 为每个列表项分配一个圆形背景的图标（深色背景，白色线条图标）。图标应与内容相关（如计算器代表数据，对话气泡代表沟通）。
    *   **文本（右侧）：** 标题加粗，下方跟随一段简短的说明文字。

2.  **视觉引导：**
    *   如果列表项可以归类，使用左侧的竖线或括号将它们分组，并添加一个数字标记（如 1, 2）。
    *   列表项之间保持足够的垂直间距。

# McKinsey 清单框 (Checklist Box)

**适用场景：**
移交清单、核查表、自测问卷。

**视觉风格参考：**
McKinsey4.pdf, Slide 80, Slide 83

**Prompt:**
请生成一个包含在矩形边框内的检查清单。

1.  **外框：** 使用一个简单的细线矩形框将内容包围起来，使之看起来像一张独立的卡片或备忘录。
2.  **标题：** 框内顶部是加粗的标题（如 "Handover checklist"），下方有一条横线分隔。
3.  **勾选框：**
    *   使用小正方形（□）或实心小方块（■）作为列表符号。
    *   内容应简练，每行一项。
4.  **布局：**
    *   可以是单列长列表。
    *   如果是详细检查（如Slide 83），可以分为两栏布局，分别列出不同类别的检查项（如 "Format checks", "Sanity checks"）。
