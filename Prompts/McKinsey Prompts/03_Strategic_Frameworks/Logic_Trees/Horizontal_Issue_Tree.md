# PPT Prompt: 结构化问题拆解逻辑树 (Issue/Logic Tree Diagram)

**类型**: 战略与框架 (Strategic Frameworks) > 逻辑树 (Logic_Trees)
**来源**: `MCK_Analysis_Part5.md` Page 51
**适用场景**: 结构化拆解核心业务问题，展示层级分解过程。

---

**Prompt 指令**:

请创建一个专业的水平分叉逻辑树，比例为16:9，由核心向分支展开。

**核心原则（防止页面破碎化）**:
*   **这是一个单一的图表，不是四个独立的卡片。** 所有的节点必须通过连接线紧密相连，形成一个整体结构。
*   **严禁**将每一层级设计成独立的大板块或迷你幻灯片。
*   **节点内容限制**: 每个方块内的文字必须精简（仅限关键词或短语，禁止长句）。

1.  **层级结构 (从左至右)**:
    *   **Level 1 (Root)**: 最左侧放置一个深蓝色方块，代表核心问题。
    *   **Level 2**: 分裂出 2-3 个中蓝色方块，代表主要维度。
    *   **Level 3**: 继续向右分裂出更多的浅蓝色方块，代表具体因素。
2.  **连接线条**:
    *   必须使用平滑的折线或曲线将父节点与子节点**物理连接**。
    *   线条粗细应从左至右逐渐变细。
3.  **色彩渐变**:
    *   方块的颜色深度应随层级增加而逐渐变浅（从深蓝到浅蓝/浅灰）。
4.  **排版要求**:
    *   方块内的文字应简洁有力，保持居中对齐。
    *   确保分支之间有足够的垂直间距，避免视觉拥挤，但**不要**让它们看起来像断开的列表。
5.  **色彩规范**:
    *   深蓝、中蓝、浅蓝。
