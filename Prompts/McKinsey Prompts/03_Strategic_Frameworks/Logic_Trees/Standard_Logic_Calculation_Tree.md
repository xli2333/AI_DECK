# PPT Prompt: 标准逻辑计算树 (Standard Logic Calculation Tree)

**类型**: 战略与框架 (Strategic Frameworks) > 逻辑树 (Logic_Trees)
**适用场景**: 利润/成本驱动因素分析、计算逻辑展示。

**视觉风格参考**: McKinsey4.pdf, Slide 68

**Prompt 指令**:

请生成一个水平方向的逻辑树（Logic Tree）或分解图。

1.  **布局方向**: 从左向右发散。
2.  **节点样式**:
    *   **根节点（左）**: 主要问题或总目标（如 "Profit"）。使用矩形框，深色边框或背景。
    *   **分支（右）**: 使用这种线条连接子节点——线条先水平延伸，再垂直分叉，最后水平连接到子节点（类似电路板走线或组织架构图线条）。不要使用斜线。
3.  **层级**:
    *   至少包含3个层级（总 -> 一级驱动因素 -> 二级细分）。
    *   例如：Profit -> (Revenue - Cost) -> (Price * Vol) - (Fixed + Variable)。
4.  **标注**:
    *   在每个节点旁边可以添加圆圈字母（a, b, c）作为索引，用于对应外部的详细说明。
    *   如果是计算逻辑，可以在节点之间添加小的数学符号圈（+,-, x）。
