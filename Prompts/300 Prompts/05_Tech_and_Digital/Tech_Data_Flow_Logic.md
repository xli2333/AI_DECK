# Slide Prompt: 数据流向与逻辑图 (Data Flow Logic Diagram)

**User Goal**: 
解释复杂系统内部的运行逻辑，如订单处理流程、数据算法模型或支付结算路径。

**Visual Layout**:
- **Flow**: 从左到右或从上到下的**逻辑路径 (Logic Path)**。
- **Components**:
  - **Input**: 数据源（文档图标）。
  - **Processing**: 处理节点（齿轮、算法芯片、服务器图标）。
  - **Decision Points**: 判断节点（菱形）。
  - **Output**: 结果（报表、终端设备）。
- **Connectors**: 使用折线或直线连接各节点，箭头指示方向。
- **Grouping**: 使用浅色背景框（Container）将相关的节点分组（如“Online System” vs “Offline System”）。

**Data & Content Guidance**:
- **Clarity**: 逻辑清晰，避免线条交叉混乱。
- **Style**: 线框图或扁平化设计，类似 Visio 或 Draw.io 风格但更美观。
